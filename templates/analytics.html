Analisis & Kesimpulan

Kelebihan Masing-masing Algoritma:
AES-256
  • Keamanan sangat tinggi (standar industri)
  • Performa cepat pada hardware modern
  • Mendukung berbagai mode operasi
  • Cocok untuk data sensitif
DES
  • Sederhana dan mudah dipahami
  • Cocok untuk pembelajaran
  • Performa cukup baik untuk file kecil
  • Sudah tidak disarankan untuk produksi
RC4
  • Sangat cepat (stream cipher)
  • Overhead minimal
  • Cocok untuk enkripsi real-time
  • Keamanan sudah dianggap lemah
Kekurangan:
AES-256
  • Membutuhkan resource lebih banyak
  • Ukuran file sedikit lebih besar (padding)
  • Implementasi lebih kompleks
DES
  • Kunci terlalu pendek (64-bit)
  • Rentan terhadap brute force
  • Tidak aman untuk data sensitif
RC4
  • Ditemukan berbagai kelemahan
